# Music Playlist Manager

---

## Класс `Node`
Базовый узел односвязного списка.  
Хранит данные одного трека и ссылку на следующий элемент.

## Класс `Music`
Основной класс плейлиста. Содержит методы для управления треками.

### `append_to_end(name, time, genre, rating)`
Добавляет новый трек в конец плейлиста.  
Сложность — **O(n)**.

### `prepend(name, time, genre, rating)`
Добавляет трек в начало плейлиста.  
Сложность — **O(1)**.

### `dell(del_name)`
Удаляет первый найденный трек с указанным именем.  
Если трек не найден, список остаётся без изменений.

### `get_set()`
Выводит в консоль все треки плейлиста с их характеристиками (название, длительность, жанр, рейтинг).

### `shak()`
Случайным образом перемешивает треки в плейлисте.  
Количество перемешиваний зависит от размера списка.

### `play_all()`
Последовательно «воспроизводит» все треки с интервалом 0.5 секунды.  
Поведение зависит от текущего режима повтора (`none`, `one`, `all`).

### `set_repeat_mode(mode)`
Устанавливает режим повтора:  
- `"none"` — без повтора  
- `"one"` — повтор текущего трека  
- `"all"` — повтор всего плейлиста после завершения  

### `safe_in_file()`
Сохраняет текущий плейлист в CSV-файл `music_save.csv`.  
Кодировка — UTF-8.

### `see_table()`
Загружает данные из CSV-файла и отображает их в виде таблицы (используется `pandas`).

### `read_csv()`
Считывает треки из CSV-файла и добавляет их в конец текущего плейлиста.

### `report()`
Выводит общую статистику:  
- Количество треков  
- Суммарное время звучания  
- Медианный (средний) рейтинг  

### `search_genre()`
Запрашивает жанр у пользователя и выводит все треки из CSV-файла, соответствующие этому жанру.

### `search_reating()`
Запрашивает минимальный рейтинг и выводит треки с рейтингом **не ниже** указанного.

# Music Playlist Manager (C++)

Консольное приложение для управления музыкальным плейлистом на основе двусвязного списка. Поддерживает добавление треков в начало и конец, удаление по названию, перемешивание, сохранение в текстовый файл и отображение содержимого с учётом ссылок на предыдущий элемент.

---

## Класс `Node`
Узел двусвязного списка.  
Хранит данные трека и указатели на следующий и предыдущий элементы.

### Поля:
| Поле | Тип | Описание |
|------|-----|----------|
| `name` | `string` | Название трека |
| `time` | `int` | Длительность в секундах |
| `genre` | `string` | Жанр трека |
| `rating` | `int` | Рейтинг (оценка) |
| `next` | `Node*` | Указатель на следующий узел |
| `previous` | `Node*` | Указатель на предыдущий узел |

---

## Класс `Music`
Основной класс плейлиста на базе двусвязного списка. Управляет узлами и операциями над ними.

### Поля:
| Поле | Тип | Описание |
|------|-----|----------|
| `head` | `Node*` | Указатель на первый элемент списка |
| `end` | `Node*` | Указатель на последний элемент списка |
| `size` | `int` | Количество треков в плейлисте |

---

### `Music()` — Конструктор
Инициализирует пустой плейлист:  
`head = nullptr`, `end = nullptr`, `size = 0`.

---

### `~Music()` — Деструктор
Освобождает память, удаляя все узлы списка.  
Предотвращает утечки памяти при уничтожении объекта.

---

### `append_to_end(name, time, genre, rating)`
Добавляет новый трек **в конец** плейлиста.
- Если список пуст — новый узел становится и `head`, и `end`.
- Иначе вставляется после `end`, обновляются указатели `previous` и `next`.
- Сложность — **O(n)** (из-за прохода до конца списка).

---

### `append_to_head(name, time, genre, rating)`
Добавляет новый трек **в начало** плейлиста.
- Если список пуст — новый узел становится и `head`, и `end`.
- Иначе вставляется перед `head`, обновляются указатели.
- Сложность — **O(1)**.

---

### `del(name)`
Удаляет трек с указанным названием.
- Если список пуст — выводит сообщение и выходит.
- Если удаляемый элемент первый — смещает `head`, обнуляет `previous` у нового `head`.
- Если удаляемый элемент последний — обновляет `end`.
- Если элемент в середине — перелинковывает соседние узлы.
- Если трек не найден — выводит сообщение об ошибке.
- Сложность — **O(n)**.

---

### `shuffle()`
Случайно перемешивает треки в плейлисте (алгоритм Фишера-Йетса).
1. Копирует все указатели узлов во временный массив.
2. Перемешивает массив.
3. Восстанавливает связи `next` и `previous` в новом порядке.
4. Обновляет `head` и `end`.
- Выводит подтверждение: `"Playlist is shuffled!"`.
- Сложность — **O(n)** по времени и памяти.

---

### `save_to_file()`
Сохраняет плейлист в текстовый файл (`table.txt`)
